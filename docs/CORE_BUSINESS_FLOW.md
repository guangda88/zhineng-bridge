# Zhineng Bridge 核心业务流程

## 1. 项目概述

**项目名称**: Zhineng Bridge (智桥)
**核心目标**: 让用户在手机上实时回复 AI 编码工具的问询，并下达指令

**支持的工具**:
- Crush AI Assistant
- Trae
- Claude Code
- Cursor
- 其他支持 WebSocket 通信的 AI 编码工具

---

## 2. 核心业务流程

### 2.1 整体架构

```
┌─────────────────┐         WebSocket         ┌─────────────────┐
│  移动端用户     │ ◄────────────────────► │  Relay Server   │
│  (手机浏览器)   │      (ws://server)      │  (Node.js)      │
└─────────────────┘                          └────────┬────────┘
                                                    │
                                                    │ WebSocket
                                                    │
                                           ┌────────▼────────┐
                                           │  AI 编码工具     │
                                           │  (CLI/Web IDE)  │
                                           └─────────────────┘
```

### 2.2 详细流程

#### 阶段 1: 连接建立

**步骤 1.1: 移动端连接**
```
用户操作:
1. 在手机浏览器打开 mobile-client.html
2. 输入服务器地址: ws://192.168.31.99:8001
3. 输入频道名称: ai-assistant
4. 点击"连接"按钮

技术实现:
- 建立 WebSocket 连接到 Relay Server
- 加入指定频道
- 接收连接确认
```

**步骤 1.2: AI 工具连接**
```
AI 工具操作:
1. 启动 AI 编码工具（如 Crush）
2. 配置 WebSocket 连接到 Relay Server
3. 连接到相同频道: ai-assistant

技术实现:
- 建立独立的 WebSocket 连接
- 加入相同频道
- 接收频道成员通知
```

#### 阶段 2: AI 工具工作

**步骤 2.1: AI 工具处理编码任务**
```
场景: 用户在电脑上使用 Crush AI Assistant 编写代码

1. Crush 分析代码库
2. Crush 发现需要用户决策的点
3. Crush 准备问询消息
```

**步骤 2.2: 发送问询到 Relay Server**
```
消息格式:
{
  "type": "command",
  "channel": "ai-assistant",
  "payload": {
    "question": "是否继续重构 userAuth 函数？",
    "options": ["继续", "跳过", "取消", "查看详情"],
    "sessionId": "session_123456",
    "timestamp": 1710000000000
  }
}

技术实现:
- Crush 通过 WebSocket 发送消息到 Relay Server
- Relay Server 广播消息到频道内所有客户端
```

#### 阶段 3: 移动端接收和响应

**步骤 3.1: 移动端接收问询**
```
用户看到:
┌─────────────────────────────┐
│  ❓ AI问询                 │
│  是否继续重构 userAuth 函数？│
│                           │
│  [继续] [跳过] [取消] [查看详情]│
└─────────────────────────────┘

技术实现:
- 移动端 WebSocket 收到消息
- 解析消息类型和内容
- 渲染 AI 问询界面
- 显示可点击的选项按钮
```

**步骤 3.2: 用户响应**
```
方式 1: 点击选项按钮
- 用户点击"继续"按钮
- 移动端自动发送响应

方式 2: 自定义输入
- 用户在输入框输入"继续，并使用 async/await"
- 点击"发送"按钮
```

#### 阶段 4: 响应传回 AI 工具

**步骤 4.1: 移动端发送响应**
```
消息格式:
{
  "type": "message",
  "channel": "ai-assistant",
  "payload": {
    "content": "继续",
    "responseType": "option",
    "sessionId": "session_123456",
    "timestamp": 1710000001000
  }
}

技术实现:
- 移动端通过 WebSocket 发送响应
- Relay Server 广播到频道
```

**步骤 4.2: AI 工具接收并执行**
```
AI 工具操作:
1. Crush 收到用户响应"继续"
2. Crush 继续 userAuth 函数重构
3. Crush 发送执行进度更新
4. Crush 可能发送新的问询
```

#### 阶段 5: 快捷指令

**步骤 5.1: 用户下达快捷指令**
```
用户界面:
┌─────────────────────────────┐
│  [▶️ 继续] [⏸️ 暂停] [⏹️ 停止] │
│  [🔄 重试] [⬅️ 上一步]      │
│  [✅ 确认] [❌ 取消]          │
└─────────────────────────────┘

用户操作:
- 点击"⏸️ 暂停"按钮
- 移动端发送暂停指令
```

**步骤 5.2: AI 工具执行指令**
```
消息格式:
{
  "type": "command",
  "channel": "ai-assistant",
  "payload": {
    "command": "pause",
    "sessionId": "session_123456",
    "timestamp": 1710000002000
  }
}

技术实现:
- AI 工具接收 pause 指令
- 暂停当前任务
- 保存执行状态
```

## 3. 关键技术实现

### 3.1 消息协议

#### 3.1.1 AI 工具 → 移动端

**问询消息**
```json
{
  "type": "command",
  "channel": "ai-assistant",
  "payload": {
    "question": "是否继续重构 userAuth 函数？",
    "options": ["继续", "跳过", "取消", "查看详情"],
    "context": {
      "functionName": "userAuth",
      "line": 123,
      "file": "src/auth.js"
    },
    "sessionId": "session_123456"
  }
}
```

**状态更新**
```json
{
  "type": "message",
  "channel": "ai-assistant",
  "payload": {
    "from": "ai",
    "content": "正在重构 userAuth 函数...",
    "progress": 45,
    "sessionId": "session_123456"
  }
}
```

#### 3.1.2 移动端 → AI 工具

**选项响应**
```json
{
  "type": "message",
  "channel": "ai-assistant",
  "payload": {
    "content": "继续",
    "responseType": "option",
    "selectedOption": 0,
    "sessionId": "session_123456"
  }
}
```

**自定义响应**
```json
{
  "type": "message",
  "channel": "ai-assistant",
  "payload": {
    "content": "继续，但使用 TypeScript 接口",
    "responseType": "custom",
    "sessionId": "session_123456"
  }
}
```

**快捷指令**
```json
{
  "type": "command",
  "channel": "ai-assistant",
  "payload": {
    "command": "pause",
    "sessionId": "session_123456"
  }
}
```

### 3.2 WebSocket 连接管理

#### 3.2.1 连接状态
- **DISCONNECTED**: 未连接
- **CONNECTING**: 连接中
- **CONNECTED**: 已连接
- **ERROR**: 连接错误

#### 3.2.2 重连机制
- 自动重连: 断开后 3 秒自动重连
- 最大重试: 5 次
- 指数退避: 每次重连间隔加倍

#### 3.2.3 心跳机制
- 客户端发送: `{"type": "ping"}`
- 服务器响应: `{"type": "pong", "timestamp": 1710000000000}`
- 间隔: 30 秒

## 6. 使用场景示例

### 场景 1: 代码重构问询

```
背景: 用户在电脑上使用 Crush 重构代码

流程:
1. Crush 分析发现多个函数需要重构
2. Crush 发送问询: "是否重构 userAuth 函数？"
3. 用户在手机上看到问询和选项
4. 用户点击"继续"按钮
5. Crush 收到响应，开始重构
6. Crush 发送进度: "重构完成 50%"
7. Crush 完成后发送: "userAuth 重构完成"

用户体验:
✅ 无需回到电脑
✅ 一键响应问询
✅ 实时查看进度
✅ 随时下达指令
```

### 场景 2: 错误处理

```
背景: Crush 执行任务时遇到错误

流程:
1. Crush 遇到错误: "模块导入失败"
2. Crush 发送问询: "如何处理导入错误？"
3. 选项: ["重试", "忽略", "修改路径", "查看错误详情"]
4. 用户点击"查看错误详情"
5. Crush 发送错误堆栈
6. 用户查看后点击"修改路径"
7. 用户输入新路径: "/src/utils/auth"
8. Crush 重新执行

用户体验:
✅ 快速了解错误
✅ 灵活选择处理方式
✅ 自定义输入支持
✅ 即时反馈结果
```

### 场景 3: 多任务管理

```
背景: Crush 同时执行多个任务

流程:
1. Crush 发送多个问询
2. 用户逐一响应
3. Crush 暂停某个任务
4. 用户下达"继续"指令
5. Crush 恢复任务
6. 所有任务完成

用户界面:
┌─────────────────────────────┐
│  任务 1: userAuth 重构     │
│  [⏸️ 暂停] [▶️ 继续] [⏹️ 停止]│
│  进度: 45%                  │
├─────────────────────────────┤
│  任务 2: API 集成          │
│  [▶️ 继续] [⏹️ 停止]        │
│  进度: 78%                  │
└─────────────────────────────┘

用户体验:
✅ 多任务并行管理
✅ 独立控制每个任务
✅ 实时查看进度
✅ 快捷操作指令
```

## 11. 总结

**核心价值**:
1. **解放双手**: 用户无需一直守在电脑前
2. **实时响应**: AI 工具的问询即时推送到手机
3. **灵活控制**: 随时随地下达指令
4. **提升效率**: AI 工具无需等待，持续工作

**技术特点**:
- WebSocket 实时通信
- 轻量级 Relay Server
- 响应式移动端界面
- 可扩展的消息协议

**适用场景**:
- 远程开发
- 离线开发
- 多任务并行
- 团队协作
